# 桌面宠物（Desktop Pet）产品策略分析报告

**日期**: 2026-04-09（初版）| 2026-04-09（P1 更新）
**阶段**: MVP (v0.4)
**分析师**: 产品评审

---

## 1. Vision

**"让专注不再孤独。"**

一只住在你屏幕上的猫，用陪伴感替代工具感，让「保持专注」这件事从自律变成一种被关心的体验。

---

## 2. Target Segments

| Segment | 规模 | 痛点强度 | 现有替代 | 优先级 |
|---------|------|---------|---------|--------|
| 知识工作者（远程/混合办公） | 大 | 高 | Forest、番茄钟、自律 | **P0** |
| 在校大学生 | 大 | 中 | 番茄钟、自习室App | P1 |
| ADHD/专注力障碍人群 | 中 | 极高 | 药物、专业工具 | P2（谨慎） |

**核心用户画像**：25-35岁，每天在电脑前6-10小时，偶尔走神但不想被冷冰冰的倒计时管着。他们需要的是一个「有温度的提醒」，而不是另一个效率工具。

**明确不服务**：追求极致效率的硬核用户（他们需要 RescueTime 那种数据驱动工具）、企业级部署需求。

---

## 3. 优点分析（做对了什么）

### A. 情绪差异化切入 — 这是最大的资产

专注力赛道已经很拥挤（Forest、番茄 Todo、Toggl），但几乎全是工具型产品——冷冰冰的倒计时、数据图表、红绿状态灯。桌面宠物用**情感陪伴**替代了**纪律约束**，这是一个真实的市场空白。

多邻国证明了"角色人设 + 情绪绑定"可以把枯燥的事情变成习惯。桌面宠物把这个逻辑嫁接到专注力上，方向是对的。

### B. 三模式设计覆盖了不同场景

- 陪伴模式：日常桌面陪伴
- 专注模式：深度工作时不想被打扰
- 知识模式：碎片复习

一个产品覆盖了「陪伴→专注→学习」三个场景，且切换成本为零（系统托盘一键切）。这在 MVP 阶段是合理的。

### C. 人设系统是核心壁垒的基础

4 种人设不只是换文案，还改了行为参数（速度、找人间隔、说话延迟、是否走向鼠标）。这意味着同一个产品对不同用户可以提供完全不同的体验，增加了匹配概率。多邻国也是这个逻辑——Lily 和 Duo 吸引不同的人。

### D. 技术门槛低、迭代快

Electron + 原生 JS，没有框架依赖，一个人就能维护。CSS 画的猫虽然简陋，但足以传达概念，MVP 阶段这是对的——不该在设计资源上过早投入。

### E. 感知系统有「活物感」

鼠标靠近瞳孔追踪、时间感知、交互记忆——这三个叠加起来，猫不再是「按脚本播放的动画」，而是「好像真的在注意我」。这是区分「桌面宠物」和「桌面玩具」的关键。

## 4. 缺点与风险（必须正视的问题）

### P0 级问题：核心价值假设未被验证

> 用户真的会因为一只猫而更专注吗？

目前没有数据支撑。产品有「陪伴感」但没有「专注力提升」的闭环。用户第1天觉得"哇好可爱"，第7天可能觉得"好烦我要关掉它"。

**具体风险**：
- 没有追踪用户实际专注时长，无法证明产品有效
- 没有用户行为数据，不知道用户用的是陪伴模式还是专注模式
- 没有留存数据，不知道 Day 7 / Day 30 留存率

**建议**：在正式发布前，找 10-20 个目标用户做 1 周的 dogfooding，核心验证"猫的出现是否真的减少了走神"。

### P0 级问题：新鲜感 ≠ 留存

| 时间 | 用户心理 | 风险 |
|------|---------|------|
| Day 1 | 哇好可爱！点一下！再点！ | 无 |
| Day 3 | 该看的基本都看了 | 文案开始重复 |
| Day 7 | 每条提醒都见过了 | 人设的新鲜感消退 |
| Day 14 | 关掉吧有点烦 | 流失 |

当前每种人设约 80-100 条文案，看似不少，但用户每天会看到 10-20 条（提醒 + 闲置 + 互动），**一周内大量重复几乎是必然的**。

更关键的是——**文案之外的"行为深度"不够**。猫只有走路、坐下、张望、打哈欠四个闲置动作，提醒只有走到中间说一句话。没有：
- 成长系统（用了30天解锁新动作）
- 收集系统（获得不同装扮/皮肤）
- 里程碑（"你已经专注了100小时！"）

**建议**：P0 里那个「角色成长系统」应该提前。哪怕只是"连续使用天数→猫的表情更丰富"，也比纯文案有效。

### P1 级问题：产品定位模糊

到底是：
- **专注力工具**（跟 Forest 竞争）？
- **桌面宠物**（跟桌宠软件竞争）？
- **情感陪伴产品**（跟 AI 陪伴竞争）？

三者都沾边但都不深。当前状态更接近"一个有趣的桌宠，碰巧能提醒你专注"。如果定位是工具，缺少数据闭环；如果定位是宠物，缺少养成深度；如果定位是陪伴，缺少对话能力。

**建议**：MVP 阶段选一个做深。建议选「**有专注力功能的桌面宠物**」——宠物是壳，专注力是锚。先用"可爱"吸引人，用"提醒"留住人。

### P1 级问题：知识模式是半成品

知识提醒模式的问题：
- 没有间隔重复算法（所有知识点轮转展示，不考虑记忆曲线）
- 没有难度分级
- 没有复习反馈（用户点"知道了"不等于记住了）
- 本质上是一个简易 Flashcard，但 Flashcard 类产品（Anki、Quizlet）已经做得很深

**建议**：要么砍掉（MVP 阶段不丢人），要么跟一个细分场景绑定（比如只做「程序员面试题」或「英语单词」，用垂直内容做差异化）。

### P1 级问题：没有「被需要」的信号

多邻国有连续打卡天数、排行榜、好友对比。Forest 有种真树。你的猫...关了就关了，没有损失感。

用户没有理由「必须回来」。这是留存最大的敌人。

**建议**：尽快加入一个**离开成本**，哪怕是最小的：
- 连续陪伴天数（"你已经陪我 7 天了"）
- 简单的喂食/互动计数（"今天你摸了我 3 次"）
- 猫的心情系统（长时间不理它→猫会变不开心，用户会有负罪感）

## 5. Strategic Trade-offs

| 选择 | 放弃 | 原因 |
|------|------|------|
| 情感陪伴型提醒 | 严格时间管理（番茄钟、时间追踪） | 赛道已有成熟玩家，差异化在于温度而非精度 |
| 本地优先（localStorage） | 云同步、多设备 | MVP 阶段降低复杂度，且桌面宠物天然是单设备场景 |
| CSS 角色 + 文案驱动 | 高质量 Lottie/Spine 动画 | 先验证「猫能不能帮人专注」，再投入视觉升级 |
| 4 种固定人设 | AI 动态生成对话 | 文案质量可控、零延迟、零 API 成本 |
| **缺少的取舍** | | |
| 未明确放弃知识模式 | 投入精力分散 | 知识模式做不深，建议砍掉或推迟到 v0.5 |
| 未做用户验证 | 凭直觉做产品 | 需要尽快找真实用户测试 |

## 6. Key Metrics

**北极星指标**：周活跃用户的日均专注提醒响应率

> 不是 DAU，不是互动次数——核心是「用户是否真的因为猫而更专注」

**输入指标**：
- 日均提醒响应率（点掉 vs 超时忽略）
- 人设切换率（用户是否在探索不同性格）
- 模式使用时长分布（陪伴/专注/知识的占比）
- Day 1 → Day 7 留存

**健康指标**：
- 单次会话平均互动次数（太高 = 玩玩具不工作，太低 = 没感情）
- 提醒频率调整率（用户是否在频繁改间隔——说明默认值不对）
- 关闭率（点了关闭按钮 = 终极流失信号）

## 7. Growth Engine

**冷启动（0→1000 用户）**：
- 小红书/抖音/B站发「桌面宠物」内容，天然有视觉传播力
- 截图传播：猫的搞笑文案/反应适合做社交货币
- GitHub 开源 + Product Hunt 发布

**增长飞轮（假设）**：
```
内容传播（猫很可爱）→ 下载试用 → 人设匹配（找到喜欢的性格）
→ 日常陪伴形成习惯 → 成长系统锁定 → UGC 分享（我的猫今天说...）
```

**当前缺口**：飞轮在「日常陪伴形成习惯」这一步断了——没有成长系统，没有离开成本。

## 9. Defensibility（诚实评估）

**当前壁垒：极弱。**

一个工程师用 Electron + CSS 画只猫 + 写几套文案，一周能做出一个功能类似的 MVP。桌面宠物这个品类本身没有技术壁垒。

**可构建的壁垒（按可行性排序）**：
1. **情感绑定**（转换成本）— 用户和猫建立了感情，换一只猫不习惯。需要成长系统+记忆系统支撑。
2. **人设数据飞轮**（数据壁垒）— 如果接入 AI，基于用户交互数据优化文案风格，越用越懂你。
3. **内容生态**（网络效应）— 用户自定义人设/皮肤/知识点并分享，UGC 内容积累。
4. **品牌**（品牌壁垒）— 「那个桌上的橘猫」变成文化符号（多邻国的 Duo 就是例子）。

## Strategic Risks（Top 3）

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| **新鲜感消退，Day 7 留存 < 20%** | 高 | 致命 | 立即加入成长系统+离开成本，在发布前验证 |
| **定位模糊导致传播失焦** | 中 | 高 | 对外统一一句话：「一只帮你专注的猫」，不要说「专注工具+桌面宠物+知识卡片」 |
| **Electron 资源占用被用户吐槽** | 中 | 中 | 优化内存占用，考虑开机自启后的常驻体验 |

## 变更日志

### 2026-04-09 — P1 优化完成 (v0.3 → v0.4)

#### 砍掉知识模式
- 删除了所有知识模式相关代码（main.js、pet.js、renderer.js、index.html、pet.css）
- 产品聚焦为「陪伴 + 专注」双模式，减少认知负担
- popIn/popOut 动画保留供专注模式使用

#### 心情系统（"被需要"信号）
- 新增 mood 数据模型：分数 0-100，下限 20，每日重置为 60
- 心情增减规则：点击 +3、接受提醒 +8、忽略提醒 -8、拖拽 +2、30 分钟无互动 -3
- 心情影响行为：
  - 高心情：行走概率 +30%、速度 +10%、眨眼更频繁、10% 自发转圈
  - 低心情：行走概率 -50%、速度 -20%、找人间隔减半（更频繁找人）
- 4 种人设各新增 moodHigh/moodLow 专属文案
- CSS 视觉反馈：低心情尾巴变慢/阴影变淡，高心情腮红加深/尾巴变快
- 设置面板新增心情显示

#### 用户行为数据采集
- 新增 analytics 系统：session 追踪，保留最近 30 条记录
- 每 60 秒记录当前模式使用时间（companion/focus）
- 追踪：互动次数、提醒接受/忽略数、模式使用时长
- 设置面板新增"今日数据"区块：提醒响应率、使用时长、陪伴/专注占比
- 关闭应用时自动保存 session
